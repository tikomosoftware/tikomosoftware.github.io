# 2 — じゃんけん（多重ディスパッチに「用途」が出た本）

コード: [`examples/rps.prism`](../../examples/rps.prism)

## これは何ができるか
じゃんけんです。あなたの手を入力すると、CPUの手と勝敗が出ます（CPUは純粋言語なので決定論的）。

```
echo rock     | python cli.py run examples/rps.prism  → you: rock / cpu: scissors / you win!
echo paper    | …                                     → you: paper / cpu: paper / draw
echo scissors | …                                     → you: scissors / cpu: rock / you lose
```

## 主役の機能：多重ディスパッチ
じゃんけんの勝敗は「**両方の手の組合せ**」で決まります（グーだけ・パーだけでは決まらない）。これはまさに
**多重ディスパッチ** ── 1つの引数の型ではなく、**複数の引数の型の組合せ**で動作が選ばれる仕組みです。

```
capability Beats for A, B          -- 2つの型 A, B にまたがる能力
  play(a: A, b: B) : Outcome
Rock provides Beats for Paper      -- (グー, パー) のときの結果 = 負け
  play(a, b)  <-  Lose
…（全9通り）
```

`play(Rock{}, Scissors{})` と書くと、**両方の型** (Rock, Scissors) を見て「Win」の実装が選ばれる。
9通りの組合せそれぞれに実装を書いておく ── これが多重ディスパッチの正体です。

## 何が面白かったか ── 「用途が無い」と言ってた機能に用途が出た
実は多重ディスパッチは、開発の途中で「**本当に必要とする例が無いから保留**」と一度書いていた機能でした
（設計ノート所見#24）。じゃんけんは、その機能のための**ぴったりの題材**。
「用途が無い」と棚上げした機能に、直感的で具体的な使い道が出た瞬間でした。

検査器がこの機能を**ちゃんと理解している**証拠も見えました ── インスタンスが型を1つしか名指さない
（`Rock provides Beats` と書く）と、「Beatsは2つの型の能力だ」と分かっていて弾きます。

## 正直なところ
小さなゲームです。でも「多重ディスパッチ」は多くの言語に無い機能で、それを**勝敗表という誰でも分かる例**で
動かせたのが見どころ。RPNが「失敗」を見せたのに対し、こちらは「型による分岐」の別の顔を見せています。
