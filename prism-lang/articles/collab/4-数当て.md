# 4 — 数当てゲーム（純粋な言語で「ループする遊び」を作る）

コード: [`examples/guess.prism`](../../examples/guess.prism)

## これは何ができるか
1〜100の数当てゲームです。推測を入力すると「too low / too high」が返り、当てるか6回外すと終わります。

```
printf '50\n25\n42\n' | python cli.py run examples/guess.prism
→ try 1: too high / try 2: too low / try 3: correct! you got it in 3 tries
```
- 数字でない入力（"abc"）には「please enter a number」と返して**やり直し**
- 6回外すと「out of tries -- it was 42」（秘密の数は決定論的に42）

## 主役の機能：再帰でループを作り、状態を持つ
Prism には**ループ構文がありません**（`for` も `while` も無い）。では「何度も推測を読む」をどう書くか?
── **再帰**です。`ask(tries)` が自分を `ask(tries + 1)` と呼び直すことで繰り返し、**試行回数 `tries` を
引数で持ち回る**ことで状態にします。

```
ask(tries) <-  show!console "try {tries} ..."  ~>  handle(tries, read!console)
handle(tries, line) <-
  parseNum(line) match
    ok n    =>  judge(tries, n)              -- 数字 → 判定
    fail _  =>  show!console "..."  ~>  ask(tries)   -- 数字でない → やり直し
```

「数字でない入力」は `parseNum` の失敗を**その場で握って**（再プロンプトに変えて）処理します ──
失敗を外に伝播させず消費する例です。

## 何が面白かったか
今回は**バグなしのクリーンビルド**でした。3本目までは題材ごとに別の穴が出ましたが、4本目は出なかった。
これも一つの結果 ── 「対話ループ＋状態＋失敗の握り」という組合せは、もう既に（自販機などで）踏まれていて
堅かった、ということです。

## 正直なところ
古典的な数当てです。でも「ループの無い言語で、再帰だけでループと状態を表現する」関数型の考え方が、
小さく体験できます。当たり前に見えて、`for` に慣れた頭には新鮮なはずです。
