# 3 — 自販機（そして字句解析のバグを見つけた本）

コード: [`examples/vending.prism`](../../examples/vending.prism)

## これは何ができるか
自販機です。硬貨を1枚ずつ入れて、商品名を打つと、おつき付きで出てきます（コーラ150/お茶100/水80）。

```
printf '100\n100\ncola\n' | python cli.py run examples/vending.prism
→ balance 0 → 100 → 200 → clunk: cola / change 50 yen
```
- お金が足りないと「30 yen short」と出して**投入を続けられる**（状態を保つ）
- 無い商品を打つと「no such item」

## 主役の機能：時間の声 `~>` と 状態機械
**時間の声 `~>`** は Prism の5本目の声 ── 「これ、その次にあれ」という**順序**を明示する記号です。
ふつうの `<-`（依存）には順序がありませんが、自販機は「**表示 → 読む → 処理**」の順番が大事：

```
run(balance) <-  show!console "..."  ~>  step(balance, read!console)
```

そして「今いくら入っているか（balance）」を**再帰で持ち回る**ことで、状態機械（投入中 → 購入）を表現します。
ループ構文も変数の書き換えも無い純粋な言語で、状態の変化を表せている、というのが見どころです。

## 何が面白かったか ── 作ったら、字句解析の本物のバグが出た
最初、自販機は**プロンプトを1回出しただけで止まりました**。原因はプロンプト文字列の中の `--`：

> `"balance ... -- coin, or item ..."` の `--` が**コメントとして剥がされ**、その後ろの
> `~> step(read!console)` ごと消えていた。だから読み込みも繰り返しもせず止まった（しかも検査は緑）。

Prism は `--` を行コメントの記号にしているのですが、字句解析が**文字列の中かどうかを見ずに**切っていた、
というバグでした。文字列を尊重するよう直し、`"a -- b"` がそのまま出るように。
長く「文字列に `--` を書くな」と注意書きしていた制約が、これで**消えました**。

これも「実際に作って動かす」ことでしか出なかった穴です（2本目のバグ発見）。

## 正直なところ
ただの自販機です。でも「順序」と「状態」を、純粋な言語で・新構文を足さずに表現できている。
RPNが「失敗」、じゃんけんが「型の組合せ」を見せたのに対し、これは「**時間と状態**」を見せています。
そして今回は、あなたが仕様（おつり・投入方法・商品）もテスト入力も決めた、一番コラボらしい回でした。
