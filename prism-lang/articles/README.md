# この言語（Prism）について、後で読み返すための記事

「自分が何を作ったのか・それがどれくらいすごいのか」が分かりにくい、という相談を受けて、
**誇張せず・でも本当にすごい所はちゃんと**まとめた読み物です。気が向いたときに上から読んでください。

1. **[01-何がすごいのか.md](01-何がすごいのか.md)**
   ── まず結論。あなたが作ったものの正体と、なぜそれが「ちゃんとした作品」なのか。
2. **[02-技術ハイライト.md](02-技術ハイライト.md)**
   ── 具体的に何を実装したか。検査器が強制していること、規模、アーキテクチャ（コードの場所つき）。
3. **[03-正直な評価.md](03-正直な評価.md)**
   ── 誇張しないための較正。「新しい所／普通の所／やらないと決めた所」。人に説明するときの言い方。
4. **[04-開発の物語.md](04-開発の物語.md)**
   ── ここまでの道のり（v0.3 → v2.0）と、印象的だった瞬間（言語が自分のバグを見つけた話など）。
5. **[collab/](collab/)**
   ── AIと一緒に作った5本（RPN電卓・じゃんけん・自販機・数当て・ランキング）の**1本ずつの振り返り**。
   それぞれ「何ができるか／どの機能が主役か／何が面白かったか（バグが出た本も）」を平易に。

> ギャラリー **300枚**／コミット **67**／タグ **v0.3〜v2.0**。すべて Python 3 のみ・依存ゼロ・ビルド不要。
