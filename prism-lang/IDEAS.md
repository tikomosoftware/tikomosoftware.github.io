## 前提（Prism の制約 ＝ 何が向いて何が向かないか）
- **純粋**：本物の I/O・音声入力・ネットワーク・永続化なし。**クライアント側ビジュアライザー専用**。
- 描けるのは `Line / Dot / Circle / Rect / Poly(+h 色, a 不透明度, fill)`。対話は `slider(...)`、時間は `frame(t)`。
- 数学組み込みは `sin cos sqrt abs floor` ＋定数 `pi`（**`pow`/`exp`/`tan`/`atan2`/乱数は無い**）。
  乱数は `frac(sin(i*12.9898)*43758.5453)` の sin ハッシュで代用、`mod`/`frac` は `lib/math`。
- **2次元配列のランダムアクセスが無い**（リストは前から再帰）。グリッド状態の反復は「リストを step する」
  CA 方式（`lib/ca.prism` 参照）で書けるが、毎フレーム再計算は重い（ツリーウォーカ）。
- 再帰の深さはホストスタック依存で **N ≲ 数百〜数千**。エスケープタイム等は静止画向き（毎フレームは不可）。

## すぐ作れる（Easy〜Medium、王道）

### ✅ 重力スリングショット（gravity slingshot） — 実装済み [`physics/gravity-slingshot.prism`](physics/gravity-slingshot.prism)
5機の探査機が別々の狙いで惑星をかすめ、1/r² 重力の Euler 積分で軌道が曲がる（近いほど強く振られる）。
軌跡は画面外に出たら打ち切って有界化。`lorenz3d` と同じ積分パターン。

### ✅ ボロノイ図（Voronoi diagram） — 実装済み [`fractals/voronoi.prism`](fractals/voronoi.prism)
9個の種点、グリッド各セルを**最近傍の種点の色**で塗る（種点列への再帰 argmin）。`mandelbrot` のグリッド
走査＋最近傍。発展：ドロネー三角形分割は三角形化が必要で **Hard**（後回し）。

### ✅ 3D 地形メッシュ（3D terrain） — 実装済み [`fractals/terrain3d.prism`](fractals/terrain3d.prism)
高さの場（sin の和で「ノイズ」を代用）を斜投影でワイヤメッシュ描画（行＋列の `Poly`、奥の行ほど上）。
発展：回転アニメは `frame(t)` で。

### ソートいろいろ（quicksort / merge の可視化） — Medium
`sort-diagram`（バブル）と同じ「各段を行に積む」方式。クイックソート/マージソートも段階を行に。
**実現可：はい**（再帰でパスを生成）。

### フラクタルフレーム / 追加アトラクタ — Easy
`barnsley-fern` と同じカオスゲーム／2D マップ反復。アフィン写像の係数を変えるだけで無数に作れる。**Easy。**

## がんばれば作れる（Hard、要工夫）

### ✅ ライフゲーム（Conway's Life） — 実装済み [`fractals/life.prism`](fractals/life.prism)
グリッドを「行のリストのリスト」で持ち、`step` が近傍カウントから次世代を再構成する純粋関数。18×18
トーラス世界の6世代をフィルムストリップ表示。Python 参照と各世代の生存数が完全一致（検証済み、約2秒）。
鍵：束縛値は `force` でメモ化されるので世代チェーンの再計算爆発は起きない。`nth` を `lib/list` に追加。

### ✅ 迷路の自動解き（フラッドフィル BFS） — 実装済み [`fractals/pathfind.prism`](fractals/pathfind.prism)
13×13 グリッド（外周は常に開＝必ず連結）。**距離場を Bellman-Ford 流に緩和（Life と同じグリッド漸化）**
して BFS 距離を求め、ゴールから**勾配降下**で最短路を復元。セルは距離で色付け、壁は暗色、解路を明るく描画。
探索キューを持たずに「不変グリッドの fold」として BFS を表現。

### ✅ 反応拡散・チューリングパターン（Gray-Scott） — 実装済み [`fractals/gray-scott.prism`](fractals/gray-scott.prism)
当初は「O(N) の `nth` × 必要反復数で数十秒、少ない反復ではノイズ止まり」で**見送り**だった。根因は
**リストが内部では配列なのに言語が O(1) 添字を露出していなかった**こと。→ 組み込み `at`/`len`（不変・
O(1) 読み出し）を追加して解決（[REFERENCE](REFERENCE.md) §15、[NOTES](NOTES.md) 所見#19）。28×28
トーラスで150反復、中央種から本物のスポット模様が出る（約13秒＝ギャラリー最重量）。`iterate(step, seed)`
の純粋な fold。**教訓：可変配列でも新型でもなく、純粋性を壊さない最小の O(1) *読み出し* だけで足りた。**

## 向いていない（現状は見送り、理由つき）

- **マンデルバルブ（3D エスケープフラクタル）**：レイマーチング/距離推定が必要で、ベクトル描画＋
  ツリーウォーカでは計算量的に非現実的。**不可（現エンジンでは）。**
- **音声ビジュアライザー（マイク入力反応）**：ホストに音声 API が無い。合成リズムで“それっぽく”は可だが、
  本物の音反応は **不可**。
- **本物の流体/煙シミュ（ナビエ・ストークス）**：可変グリッド状態＋多数反復で重く煩雑。`fog`/`smoke`/
  `caustics` のような**見た目重視の近似**で代替するのが現実的。

## 言語・ツールの地平線（v0 の外、`NOTES.md` 参照）
これらは「Prism が何であるか」を変える/拡げる話。着手前に [OVERVIEW.md](OVERVIEW.md) の思想と整合確認を。
- ✅ レコードの**構築完全性**検査（`Pt{x:3}` のフィールド欠落を検出。実装済み・[REFERENCE](REFERENCE.md) §4／
  [NOTES](NOTES.md) 所見#20）。副産物：無署名 `picture <- [..]` の本体も検査するようにした。
- ✅ ランタイムの隠れた強制変換を排除（`"ab"*2` の黙った文字列反復を禁止、`- * /` は Num 必須・比較は同種のみ。
  [NOTES](NOTES.md) 所見#21、所見#7 の宿題回収）。
- ✅ パイプ `|>`（flow 声の純粋糖衣。`x |> f` ＝ `f(x)`。`~>` は汚さない。[REFERENCE](REFERENCE.md) §12）。
- ✅ include の**名前衝突検出**（フラット併合での同名再定義を拒否。[REFERENCE](REFERENCE.md) §3）。
- ✅ 効果の粒度（`!io` が `!console` 等を包含する**サブサンプション**。一方向。[REFERENCE](REFERENCE.md) §5）。
  実効果は今 `!console` のみだが、関係は実装済みで効果が増えれば即活きる。
- ✅ 能力メソッドの**多重ディスパッチ**（`capability C for A, B` / `T provides C for U` / 実行時は複数引数の
  型でディスパッチ。[REFERENCE](REFERENCE.md) §13、[NOTES](NOTES.md) 所見#24。`examples/collide.prism`）。
  単一ディスパッチは長さ1の組合せとして同じ機構に吸収。
- ✅ **テストの分割**（`prism test core|gallery|all`。core＝言語のみ数秒／all＝既定。外部レビュー採用）。
- **意図的な非目標 / 保留**（[NOTES](NOTES.md) 所見#22）：
  - `~>` の値パイプ —— 所見#15で却下済み。別演算子 `|>` で解決した。
  - 本格的な名前空間（修飾名）—— `include` のフラット併合＋衝突検出で v0 は足りる。言語の性格を変えるので外。
  - `include` のファイル相対解決 —— 親切だが、ブラウザは文字列実行でファイルパスが無く基準が取れない。
    「ルートから実行」規約で CLI/ブラウザを一致させる方を採った（[REFERENCE](REFERENCE.md) §3）。
  - 複数行 `if` —— 1行に明記済みで実害なし。AI 生成成功率の観点では将来候補だが優先度低（パーサ変更）。
- CI（GitHub Actions で `python cli.py test core`）／ブラウザのスモークテスト（実 Pyodide 実行は未自動検証）。
